# Lab2多次填充攻击流密码实验报告

## 1. 实验原理
流密码的加密核心是明文与伪随机密钥流的异或运算：C=M⊕PRG(k)。若同一密钥被重复使用，攻击者可将两段密文异或，消去密钥流得到明文异或结果：C1​⊕C2​=M1​⊕M2​。
利用空格（0x20）与字母异或的特性（大写↔小写），可推断明文中的空格位置：

若 M1​⊕M2​=0x20 → 其中一个是空格，另一个是字母；
若 M1​⊕M2​=0 → 两个位置均为空格；
结合可打印字符范围（0x20~0x7E）验证密钥流的合理性。

## 2. 实现思路
### 2.1 核心步骤

格式转换：将十六进制密文转换为字节数组，便于异或操作；
空格推断：遍历所有密文的相同位置，通过两两异或结果识别空格候选位置；
密钥流还原：基于空格（0x20）推断对应位置的密钥流，并验证其在所有密文中的合理性；
补全密钥：对未推断出的密钥位置，选择能让所有密文对应位置生成可打印字符的密钥；
明文还原：用密钥流与密文异或，得到最终明文。

### 2.2 关键逻辑

异或操作：逐字节对密文进行异或，核心函数 xor_bytes；
空格检测：通过 Ci​⊕Cj​ 是否为 0x20/0 判断空格候选；
密钥验证：确保密钥生成的明文均为可打印字符（0x20~0x7E）。

## 4. 验证方法

字符合理性：解密后的明文均为可打印 ASCII 字符，无乱码；
语义验证：目标密文解密结果为通顺的英文句子，符合自然语言特征；
交叉验证：用还原的密钥流重新加密明文，可还原出原始密文，证明密钥流正确。

## 5. 实验结论

流密码的安全性完全依赖 “密钥仅使用一次” 的前提，重复使用密钥会导致明文被轻易破解；
利用空格与字母的异或特性，可高效推断明文结构，进而还原密钥流；
即使仅掌握密文，攻击者也可通过统计分析和字符特征推断出核心信息（如空格、字母位置）。
