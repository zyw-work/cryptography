## 第 4 题

回顾课程中讨论的 Luby-Rackoff 定理：对安全的 PRF 应用**三轮** Feistel 网络可以得到安全的分组密码。现在让我们看看如果只使用**两轮** Feistel 会出什么问题。

设 $F: K \times \{0,1\}^{32} \to \{0,1\}^{32}$ 是一个安全的 PRF。两轮 Feistel 网络定义了以下 PRP $F_2: K^2 \times \{0,1\}^{64} \to \{0,1\}^{64}$：

$$
\begin{aligned}
\text{输入} &: (L_0, R_0) \\
\text{第1轮} &: L_1 = R_0, \quad R_1 = L_0 \oplus F(k_1, R_0) \\
\text{第2轮} &: L_2 = R_1 = L_0 \oplus F(k_1, R_0), \quad R_2 = L_1 \oplus F(k_2, R_1) = R_0 \oplus F(k_2, R_1) \\
\text{输出} &: (L_2, R_2)
\end{aligned}
$$


以下四组数据中，一组是使用随机密钥的 PRP $F_2$ 的输出，而其他三组是真正随机排列 $f: \{0,1\}^{64} \to \{0,1\}^{64}$ 的输出。所有 64 位输出均编码为 16 个十六进制字符。你能判断哪一组是 PRP 的输出吗？

> **提示**：首先证明 $F_2(\cdot, 0^{64})$ 与 $F_2(\cdot, 1^{32}0^{32})$ 的异或中存在可检测的模式，然后尝试在给定的输出中检测该模式。

- [ ] 输入 $0^{64}$ 时输出为 `5f67abaf5210722b`；输入 $1^{32}0^{32}$ 时输出为 `bbe033c00bc9330e`
- [ ] 输入 $0^{64}$ 时输出为 `4af532671351e2e1`；输入 $1^{32}0^{32}$ 时输出为 `87a40cfa8dd39154`
- [ ] 输入 $0^{64}$ 时输出为 `2d1cfa42c0b1d266`；输入 $1^{32}0^{32}$ 时输出为 `eea6e3ddb2146dd0`
- [x] 输入 $0^{64}$ 时输出为 `9f970f4e932330e4`；输入 $1^{32}0^{32}$ 时输出为 `6068f0b1b645c008`

## 第 5 题

**基于 Nonce 的 CBC 模式**

回顾课程中讨论的内容：如果要在 CBC 加密中使用非随机的唯一 nonce，则必须先使用一个**独立的** PRP 密钥对 nonce 进行加密，然后将结果作为 CBC 的 IV 使用。现在让我们看看如果使用**相同的** PRP 密钥来加密 nonce 和进行 CBC 加密会出什么问题。

设 $F: K \times \{0,1\}^\ell \to \{0,1\}^\ell$ 是一个安全的 PRP（令 $\ell = 128$）。设 $n$ 为一个 nonce，对消息 $m$ 的加密过程为：先计算 $IV = F(k, n)$，然后使用 $F(k, \cdot)$ 进行 CBC 加密（使用该 IV）。注意这里使用的是同一个密钥 $k$。

攻击者首先请求加密双分组消息 $m = (0^\ell, 0^\ell)$（nonce $n = 0$），收到双分组密文 $(c_0, c_1)$。根据 CBC 的定义，我们知道 $c_1 = F(k, c_0)$。接下来，攻击者请求加密单分组消息 $m_1 = c_0 \oplus c_1$（nonce $n = c_0$），收到单分组密文 $c_0'$。

请问 $c_0$、$c_1$、$c_0'$ 之间满足什么关系？（该关系使得攻击者能以优势 1 赢得基于 nonce 的 CPA 游戏）

- [ ] $c_1 = c_0$
- [ ] $c_0 = c_1$
- [x] $c_1 = c_0'$
- [ ] $c_0 = c_0'$

## 第 27 题

在实际使用随机化加密（如 CBC with random IV、CTR with random nonce）时，需要特别注意哪些问题？（可能有多个正确答案）

- [x] **随机数来源必须是密码学安全的伪随机数生成器（CSPRNG）**：使用 `rand()`、时间戳等非密码学随机源生成 IV/Nonce 可能导致可预测，破坏语义安全
- [x] **IV/Nonce 的重用会灾难性地破坏安全性**：在 CTR 模式下，两次使用相同 Nonce + Key 加密不同明文，攻击者可直接异或两段密文消去密钥流
- [x] **随机化加密本身不提供完整性和认证保护（仅提供机密性）**：攻击者可以任意篡改密文，接收方无法检测，应额外使用 MAC 或 AEAD 模式（如 AES-GCM）
- [x] 随机化加密中密钥可以重复使用多次，不影响安全性
- [x] **密文会比明文稍长**：需要将 IV 或 Nonce 一并传输给接收方（通常附在密文开头），因此密文长度 = IV/Nonce 长度 + 加密后密文长度
