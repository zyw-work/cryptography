# Lab2 实验报告：多次填充攻击流密码

## 一、使用的分析方法

本次实验采用**多次填充攻击（Many-Time Pad Attack）**，利用同一密钥多次加密带来的安全漏洞。分析方法包括以下步骤：

1. **密文异或消除密钥**  
   将不同密文两两异或，得到对应明文的异或结果：  
   \( C_i \oplus C_j = M_i \oplus M_j \)。  
   由于密钥流被消除，攻击者可直接分析明文之间的异或关系。

2. **空格模式识别**  
   英文文本中空格字符（`0x20`）频繁出现，且与字母异或时会改变字母大小写：  
   - `空格 ⊕ 大写字母 = 对应小写字母`  
   - `空格 ⊕ 小写字母 = 对应大写字母`  
   通过观察 \( M_i \oplus M_j \) 中每个字节，若该字节异或值符合空格与字母的组合模式，则可推断其中一段明文在该位置为空格，另一段为字母。

3. **密钥流恢复**  
   一旦推断出某位置的明文字符，即可通过 `key = C ⊕ M` 得到该位置的密钥字节。  
   将多个密文段在同一位置的信息综合起来，利用共识机制（如投票）确定最可能的密钥字节，并迭代更新其他位置的明文。

4. **迭代推断与完善**  
   在初始推断的基础上，逐步扩大已知明文的范围，利用已恢复的密钥字节反推其他位置的明文，再通过新的明文推断更多密钥字节。经过多轮迭代，最终恢复完整的密钥流。

5. **目标解密**  
   将恢复的密钥流与目标密文异或，得到目标明文。

## 二、如何确认目标密文的明文内容

为确保解密结果的正确性，采用以下验证方法：

1. **可读性检查**  
   解密后的字节序列应全部为可打印 ASCII 字符（32–126），且构成符合英语语法和拼写的句子。如果出现大量不可打印字符或随机字节，说明密钥流恢复存在错误。

2. **交叉验证**  
   使用恢复的密钥流解密其他已知密文（如密文 #1 至 #10），检查是否得到合理的英文文本。若所有密文解密后均呈现可读内容，则证明密钥流准确。

3. **统计一致性**  
   密钥流本身应呈现伪随机特性，其字节分布无明显规律，各字节值在 0–255 之间均匀分布。

4. **人工审阅**  
   最终明文应包含有意义的英文句子，与题目背景（流密码安全）相关，且逻辑连贯。
